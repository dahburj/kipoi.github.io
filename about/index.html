What is Kipoi?

Kipoi is a repository containing hundreds of predictive models for genomics. It provides unified means to access and use these models. Each model comes with the code to load and preprocess data. This greatly facilitates applying models to new data and creating new models.

What can I do with Kipoi?

With Kipoi you can:

  • Explore hundreds of trained models for genomics and apply them to new data in few lines of code in
    • python
    • R
    • command line.
  • Evaluate the impact of genetic variants with the majority of Kipoi models. See docs and notebook.
  • Fine-tune an existing model on a new dataset. See our notebook and Keras fine-tuning example.
  • Compose a new model using other models as building blocks. See notebook.
  • Publish your own models and reach a broad community. See how.

Machine learning frameworks are easily integrated into Kipoi. So far we integrated:

  • Tensorflow
  • Pytorch
  • Keras
  • Scikit-learn
Models not developed with one of these frameworks can also be submitted to Kipoi. See how.

Why do we need it?

In genomics, we aim to understand how genetic sequence - a text of 3 billion A’s, C’s, G’s, and T’s - encodes biological processes and predisposes to diseases. Worldwide, research labs are deciphering individual aspects of this code by training predictive models to data produced by high-throughput technologies such as sequencing and genome-wide CRISPR-Cas. However, these predictive models are implemented in various programming languages and machine learning frameworks. They are made available through different channels, such as code repositories and supplementary material of articles. This heterogeneity makes it difficult to apply models to new data, to compare models, and to re-use models while developing new ones. By providing a unified access to models, we want to lower the entry barrier to modeling in genomics and to accelerate the translation of basic genomics research into medical applications.
